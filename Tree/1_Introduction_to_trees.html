Introduction to trees

Tree: It is a non-linear data structure which organizes data in hierarchial structure recursively
  • In a tree every individual element is called node.
  • In a tree with n nodes there will be n-1 edges.

Tree Terminologies

  • Root: First node of the tree.
  • Edge: Connecting link between two nodes.
  • Parent: Tree node which has child/children, ie. any node which is predecessor to any other node.
  • Child: Descendent of any node.
  • Siblings: Nodes with same parent.
  • Leaf: Nodes which do not have children, also called the terminal/external nodes.
  • Internal Nodes: Nodes with atleast one child.
  • Degree: Number of children a node has.
  • Level: Root node is at level 0, its children on level 1 and so on...
  • Height: Total number of edges from leaf to the node in longest path. Height of leaf nodes is 1. Height of root node is height of tree that is number of edges from farthest leaf node to root + 1.
  • Depth: Total number of edges from root to that node. Depth of root node is 1. Depth of tree is number of edges from root to farthest leaf node + 1.
  • Path: Sequence of nodes and edges between two nodes.
  • Diameter: Longest path between two nodes. Path does not need to pass via root. It is sum of height of the left subtree and height of right subtree.
  • Sub-tree: Tree formed by a node and its children recursively
  • Ancestor: Nodes occuring in path to root.
  • Full Binary Tree: Every node has either 0 or 2 children.
  • Complete Binary Tree: All levels are completely fiilled except the last level. The last level has all nodes on left as possible.
  • Perfect Binary Tree: All left nodes are on the same level.
  • Balanced Binary Tree: Height of tree should be at maximum log(n) where n is number of nodes.
  • Degenerate Tree: Skewed tree which is not balanced.
